# Decisions

## D001: Build the first experience as a real landing page

The first screen should be the usable Lancotech marketing website, not a
template placeholder or auth-first app. The user needs immediate confidence in
the product direction and brand.

## D002: Keep Convex installed but do not require auth for the landing page

The existing project includes Convex Auth. The public landing page should render
without sign-in because its purpose is acquisition, education, and product
positioning.

## D003: Use custom dashboard mockups instead of stock images

The requested hero needs trading dashboards, charts, analytics, portfolio
tracking, AI assistant, and learning progress. Custom UI mockups communicate the
product more directly than generic stock media.
