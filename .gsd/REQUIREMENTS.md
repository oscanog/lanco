# Requirements

## Product Requirements

- The first public experience must be a mobile-first landing page, not a
  starter template or marketing-only placeholder.
- The page must immediately communicate: Learn. Practice. Master. Trade with
  Confidence.
- The hero must include a large headline, supporting copy, primary CTA
  `Start Free`, secondary CTA `Watch Demo`, trust badges, and a custom trading
  dashboard illustration.
- The page must include sections for features, why simulation matters,
  dashboard preview, learning journey, testimonials, statistics, pricing, FAQ,
  final CTA, and footer.
- The feature model must cover Learn, Simulate, Track, and Improve.
- Pricing must include Free, Premium, and Enterprise, with Free highlighted.
- The page must avoid gambling, hype, and crypto-scam visual language.

## Technical Requirements

- Use React and Vite in the existing app.
- Use Framer Motion for fade, slide, scale, hover, parallax, and reveal
  animations.
- Use reusable components for buttons, cards, feature cards, pricing cards,
  navbar, footer, dashboard cards, stats, testimonials, FAQ accordion, timeline,
  badges, charts, and callouts.
- Use mobile-first responsive layout.
- Use the exact palette from `.gsd/PROJECT.md`.
- Run `npm run build` before milestone closeout.

## Accessibility Requirements

- Navigation and FAQ controls must be keyboard-accessible buttons or links.
- Text must stay readable on mobile and desktop.
- Decorative visuals must not block core content.
- CTA labels must be explicit and scannable.
